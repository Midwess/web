The network handler is mounted at `/__worldant/v1`.

## Value call

    <DBox badge={3} kind="success" title="200 or 202" sub="returnValue if done; runId if still running" />

  </DRow>

`worldant-client` generates a run id for every `call()`. If the POST fails because the network drops or the world restarts, it retries with that same id. The server accepts conflicts for the same run id and converges on the existing run.

## Stream call

```ts
for await (const item of client.stream("trackOrder", [orderId])) {
  console.log(item);
}
```

The client starts a run, then opens:

```txt
GET /__worldant/v1/runs/:runId/stream
```

Each SSE data frame is base64 payload encoded by the server and decoded by the client.

## Registry endpoints

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/registry` | Contract JSON with `ETag`. |
| `GET` | `/registry.d.ts` | Type declarations. |
| `GET` | `/contract/:etag.tgz` | Generated package tarball. |
| `GET` | `/npm/:name` | npm packument for the world id. |

The handler derives package base URL from forwarded headers or `Host`.
