# Cross-origin remote functions

When the consumer runs on a different origin from the world — e.g. a Vite app on `:5173` calling a NestJS world on `:3000` — you need two things: CORS on the server, and an `.npmrc` entry that routes the world's npm scope to its registry.

## Installation (consumer)

```sh
npm install worldant-client
```

The world's typed package (`@fxdesk/quotes`, `orders`, etc.) is installed from the world's live registry — see step 3 below.

## 1. Start the world with CORS

On the server, pass `cors` with the origins you want to allow and `useOnPremisePort: true` to mount the world's routes on your app's existing HTTP server instead of spinning up a separate port:

```ts
// world.bootstrap.ts (NestJS, Express, Fastify, …)
import { createWorld } from '@midwess/worldant';

await createWorld({
  id: '@fxdesk/quotes',
  net: {
    useOnPremisePort: true,
    cors: ['http://localhost:5173', 'https://app.fxdesk.io'],
  },
});
```

`useOnPremisePort` registers `/__worldant/v1/*` on the same HTTP server your app already listens on — no extra port to manage.

## 2. Route the npm scope in `.npmrc`

Scope the world id so your package manager routes only that scope to the world's registry. Everything else — `react`, `vite`, etc. — still comes from npm:

```ini
# client/.npmrc
@fxdesk:registry=http://localhost:3000/__worldant/v1/npm
```

For production:
```ini
@fxdesk:registry=https://api.fxdesk.io/__worldant/v1/npm
```

## 3. Install the typed package

```sh
pnpm install   # @fxdesk/quotes fetched from the world, everything else from npm
```

Or explicitly:
```sh
npm install @fxdesk/quotes
```

## 4. Call from the browser

```ts
import { quote, watchRate } from '@fxdesk/quotes';

async function getQuote() {
  const lines = await quote(100);
  console.log('Quote lines:', lines);         // typed, Date fields intact

  for await (const tick of watchRate('EUR')) {
    console.log('EUR rate:', tick.rate);       // live SSE stream
  }
}
```

`defaultConfiguration.baseUrl` is baked into the generated package from the URL you installed from, so cross-origin fetch works automatically once CORS is enabled.

## Auth token

To require authentication, add `token` when creating the world:

```ts
await createWorld({
  id: '@fxdesk/quotes',
  net: {
    useOnPremisePort: true,
    cors: ['https://app.fxdesk.io'],
    token: process.env.WORLD_TOKEN,
  },
});
```

Consumers supply the token at runtime:

```ts
import { configure } from '@fxdesk/quotes';
configure({ token: import.meta.env.VITE_WORLD_TOKEN });
```

The token is sent as `Authorization: Bearer <token>` on every request and is never baked into the generated package.
