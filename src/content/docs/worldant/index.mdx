Worldant is a durable `@workflow/world` implementation for Node apps. Add `@midwess/worldant`, keep writing `"use workflow"` and `"use step"`, and store runs, events, hooks, queues, and streams in an embedded SQLite world instead of an external service.

It has two parts:

- `@midwess/worldant` opens the native world, registers it with `@workflow/core`, and optionally serves remote-function endpoints.
- `@midwess/worldant-cli` supervises the app process, owns public sockets, forwards traffic, restarts, drains redeploys, and can scale the app process to zero.

## Start here

Install the world in application boot code before workflow handlers run:

```ts
import { install } from "@midwess/worldant";

await install();
```

Run the app under the supervisor when you want socket ownership, process restart, redeploy drain, and scale-to-zero:

```bash
npm install @midwess/worldant
npm install -g @midwess/worldant-cli
worldant run -- npm start
```

## What Worldant provides

- Durable workflow runs on local disk.
- Step memoization after committed completion, so restart recovery does not repeat completed side effects.
- Queue dispatch back into your app through `@workflow/core`.
- Remote functions over `/__worldant/v1`.
- A live npm registry for generated typed clients.
- Next.js helper `withWorldant()` for workflow discovery and client generation.
- Supervisor commands for process lifecycle.

## Packages

| Package | Use it for |
| --- | --- |
| `@midwess/worldant` | Library entrypoint: `install`, `createWorld`, `getWorld`, `setWorld`, `outputStream`. |
| `@midwess/worldant/next` | Next.js config wrapper that writes `worldant.catalog.json` and optional `worldant/client.ts`. |
| `worldant-client` | Browser-safe transport used by generated client packages. |
| `worldant-react` | `useWorldant(schema, overrides)` helper for generated schema packages. |
| `@midwess/worldant-cli` | `worldant` process supervisor. |
