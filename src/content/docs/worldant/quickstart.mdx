---
title: Quickstart
description: The intended first Worldant flow.
---

Worldant is not stable yet. This quickstart describes the accepted simplified
target flow; the runtime migration is not shipped yet.

Initialize a guided blank world:

```bash
worldant init my-world --app chess
cd my-world
```

The generated world is ready to build and serve:

```text
my-world/
  worldant.json
  apps/chess/
    migrations/
    commands/
    workflows/
      steps/
    lib/
    tests/
```

Build and run:

```bash
worldant build
worldant serve
```

Consume it from any client:

```ts
import { createWorld } from "worldant-client";

const world = createWorld({ url: "ws://127.0.0.1:8787/__worldant/v1/ws" });

await world.chess.move({ gameId, from: "e2", to: "e4" }, { key: "move:e2:e4" });
const board = await world.chess.board({ gameId });
```

Both `move` and `board` are Commands. Use an explicit database transaction in a
Command when several writes must be atomic. Use a Workflow for work that must
survive retries, waiting, process failure, or restart.
