# Installation

Worldant ships as `@midwess/worldant` on npm with a prebuilt native binary for your platform. No compiler toolchain, no Rust install.

## Install

```bash
pnpm add @midwess/worldant
# or
npm install @midwess/worldant
# or
yarn add @midwess/worldant
```

This pulls in `@workflow/core`, `@workflow/world`, and a prebuilt `world.<target>.node` binary that matches your OS and architecture.

## Supported platforms

The package ships prebuilt binaries for:

| Platform | Architecture | Status |
|----------|--------------|--------|
| macOS | Apple Silicon (`arm64`) | prebuilt |
| macOS | Intel (`x64`) | prebuilt |
| Linux | `x64` (glibc) | prebuilt |

Other platforms (Linux ARM, musl libc, Windows, Deno) are not yet published as prebuilt binaries. The source build path lives in the [worldant repo](https://github.com/Midwess/worldant) on GitHub.

## Requirements

- **JavaScript.** Node.js 22+ and any package manager (`pnpm`, `npm`, or `yarn`).

No C compiler. No Rust. No system SQLite. The native binary ships prebuilt inside the package.

## Verify

```bash
node -e "import('@midwess/worldant').then(m => m.install()).then(w => console.log('ok'))"
```

You should see `ok`. This opens the default data dir (`./.world` relative to `process.cwd()`), starts the world, and registers it as the active `@workflow/core` runtime.

## Optional: install the supervisor

The `worldant` CLI is a separate package. It is only needed if you want the scale-to-zero supervisor (otherwise your app runs as a normal long-lived Node process).

```bash
pnpm add -g @midwess/worldant-cli
# or
npm install -g @midwess/worldant-cli
```

The npm package picks the right prebuilt binary for your OS via platform-specific optional dependencies — no separate install step.
