---
title: React hook
description: Use worldant-react to bind a generated schema package to a memoized client.
---

`worldant-react` exports `useWorldant(schema, overrides)`. It reads the generated package's `defaultConfiguration`, merges runtime overrides, and returns a memoized `worldant-client`.

```tsx
"use client";

import * as orders from "@acme/orders";
import { useWorldant } from "worldant-react";

export function CheckoutButton({ order }: { order: orders.Order }) {
  const client = useWorldant(orders, { token: "session-token" });

  return (
    <button
      onClick={async () => {
        await client.call<typeof orders.placeOrder>("placeOrder", [order]);
      }}
    >
      Place order
    </button>
  );
}
```

## Override fields

The hook accepts the same connection fields as `createClient()`:

| Field | Purpose |
| --- | --- |
| `baseUrl` | Remote world origin. |
| `prefix` | Endpoint prefix, default `/__worldant/v1`. |
| `token` | Bearer token. |
| `pollMs` | Poll interval after a call returns `202`. |
| `worldId` | Optional client-side label. |
| `store` | Run id persistence store. |
| `retryBaseMs` | Initial network retry delay. |
| `retryMaxMs` | Maximum network retry delay. |

Use generated wrappers for strongest types; use `client.call` or `client.stream` when building a generic UI.
