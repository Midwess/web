# Installation

Worldant ships as `@midwess/worldant` on npm with a prebuilt native binary for your platform. No compiler toolchain, no Rust install.

## Install

```bash
pnpm add @midwess/worldant
# or
npm install @midwess/worldant
# or
yarn add @midwess/worldant
```

This pulls in `@workflow/core`, `@workflow/world`, and the prebuilt native addon. The library bundles **all** of its prebuilt binaries directly in the package — its `files` field is `["dist", "native"]`, so every supported `world.<target>.node` is shipped inside the single npm tarball and the loader picks the one matching your OS and architecture at runtime. (The matching-binary-via-optional-dependencies mechanism described below applies only to the CLI package, not the library.)

## Supported platforms

The package ships prebuilt binaries for:

| Platform | Architecture | Status |
|----------|--------------|--------|
| macOS | Apple Silicon (`arm64`) | prebuilt |
| macOS | Intel (`x64`) | prebuilt |
| Linux | `x64` (glibc) | prebuilt |

Other platforms (Linux ARM, musl libc, Windows, Deno) are not yet published as prebuilt binaries. The source build path lives in the [worldant repo](https://github.com/Midwess/worldant) on GitHub.

## Requirements

- **JavaScript.** Node.js 18+ and any package manager (`pnpm`, `npm`, or `yarn`). The CLI package declares `node >= 18`; the example app uses Node 20+.

No C compiler. No Rust. No system SQLite. The native binary ships prebuilt inside the package.

## Verify

```bash
node -e "import('@midwess/worldant').then(m => m.install()).then(w => console.log('ok'))"
```

You should see `ok`. Called with no config, `install()` opens an **in-memory (ephemeral) World** via `open_temp()` — no on-disk data dir is created — then starts the world and registers it as the active `@workflow/core` runtime. To get a durable on-disk database, pass `dataDir` to `install()`/`createWorld()`. (For reference, the example app resolves its own default path as `process.env.WORLDANT_DATA_DIR ?? join(process.cwd(), '.worldant-data')` — that is an app-level default, not a library default.)

## Optional: install the supervisor

The `worldant` CLI is a separate package. It is only needed if you want the scale-to-zero supervisor (otherwise your app runs as a normal long-lived Node process).

```bash
pnpm add -g @midwess/worldant-cli
# or
npm install -g @midwess/worldant-cli
# or, build from the published crate
cargo install worldant-cli
```

The npm package picks the right prebuilt binary for your OS via platform-specific optional dependencies (`@midwess/worldant-cli-darwin-arm64`, `-darwin-x64`, `-linux-x64-gnu`) — no separate install step. The published crate is `worldant-cli`; the installed binary is named `worldant`.
