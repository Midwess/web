PgPaw is a small HTTP server that accepts read-only PostgreSQL `SELECT` queries, runs them against an embedded pglite-rs logical replica, and returns either a cacheable snapshot URL or a live SSE stream.

## Core behavior

- `POST /query` accepts a single read-only `SELECT`.
- Public queries return `303 See Other` to `/q/{hash}/{version}`.
- Private queries run under a JWT-derived Postgres role and return inline `200` JSON with `private, no-store`.
- `POST /query?live=true` streams a snapshot and row deltas.
- `GET /q/{hash}/{version}` reads from an in-process result cache and is public-cacheable for 72 hours.
- `GET /healthz` reports replica state and watermark.

## Main components

| Component | Purpose |
| --- | --- |
| `ReadClassifier` | Parses SQL, rejects unsafe statements, extracts tables and equality filters. |
| `VersionIndex` | Tracks LSN watermarks per table/key from committed replication transactions. |
| `QueryCache` | Stores JSON snapshot bodies keyed by `{hash}:{version}`. |
| `LiveHub` | Re-runs subscribed queries after commits and emits insert/update/delete/up-to-date events. |
| `Verifier` | Validates HS256 or RS256/ES256 JWTs and maps a claim to a Postgres role. |
