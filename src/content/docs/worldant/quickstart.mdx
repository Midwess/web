---
title: Quickstart
description: Add a durable world, define a workflow, and run it under Worldant.
---

## 1. Install

```bash
npm install @midwess/worldant workflow
npm install -g @midwess/worldant-cli
```

## 2. Create the world

Call `install()` before serving requests:

```ts
import { install } from "@midwess/worldant";

await install({
  id: "orders",
  dataDir: "./.worldant/orders"
});
```

`id` is required when the app runs under the supervisor and Worldant cannot derive one from package metadata.

## 3. Write workflow code

```ts
async function charge(orderId: string) {
  "use step";
  return { ok: true, orderId };
}

export async function checkout(orderId: string) {
  "use workflow";
  return charge(orderId);
}
```

Worldant implements the world interface. Workflow transforms and framework adapters still come from the Vercel Workflow package stack.

## 4. Run supervised

```bash
worldant run -- npm start
```

Run it again after a deploy:

```bash
worldant run -- npm start
```

The daemon treats the second run as the same app when the world id matches. It starts the new process, drains the old process, and keeps the public port owned by the supervisor.

## 5. Inspect

```bash
worldant ps
worldant logs orders -f
worldant stop orders
```
