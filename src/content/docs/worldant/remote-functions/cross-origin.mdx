---
title: Cross origin
description: Serve the generated package from one origin and call it from another.
---

Cross-origin mode is useful when the workflow world and browser app are separate deployments.

## Server world

```ts
import { createWorld } from "@midwess/worldant";

const world = await createWorld({
  id: "@fxdesk/quotes",
  net: {
    useOnPremisePort: true,
    cors: ["http://localhost:5173"]
  }
});
await world.start?.();
```

Use a scoped package id when you want npm to route only that scope to the world registry.

## Client install

`.npmrc`:

```ini
@fxdesk:registry=http://localhost:3000/__worldant/v1/npm
```

Install normally:

```bash
pnpm install @fxdesk/quotes
```

## Browser usage

```ts
import { quote, watchRate, configure } from "@fxdesk/quotes";

configure({ token: sessionToken });

const q = await quote(100);

for await (const tick of watchRate("EUR")) {
  console.log(tick);
}
```

The generated package's `defaultConfiguration.baseUrl` is derived from the registry URL used during installation. Override it with `configure({ baseUrl })` when a proxy or environment URL differs.
