---
title: Supervisor
description: Planned daemon and execution supervision model.
---

Worldant supervision is being redesigned around the daemon.

The daemon owns:

- build/run requests,
- `.dist` snapshot activation,
- PgPaw sessions,
- JavaScript isolate scheduling,
- workflow queues,
- reactive subscriptions,
- az-wire / MCP / HTTP endpoints,
- knowledge text endpoints,
- crash recovery.

The daemon should keep memory proportional to active work, not total generated
apps, total files, or dormant workflow runs.

```text
active calls + workflow replays + subscriptions + warm cache budget
```

Idle entries are offloaded to `.dist` and PgPaw metadata.
