---
title: Quickstart
description: Prepare upstream Postgres, start PgPaw, and issue snapshot and live queries.
---

## 1. Prepare upstream

```bash
pgpaw init \
  --pg-host 127.0.0.1 \
  --pg-port 5432 \
  --pg-user postgres \
  --pg-password "$POSTGRES_PASSWORD" \
  --pg-database myapp
```

`init` checks logical replication settings, creates the publication when missing, and installs a DDL event trigger when permissions allow it.

Restart Postgres if `wal_level=logical` or sender/slot settings changed.

## 2. Serve

```bash
pgpaw serve \
  --host 127.0.0.1 \
  --port 8080 \
  --data-dir ./cache-data \
  --pg-host 127.0.0.1 \
  --pg-user postgres \
  --pg-password "$POSTGRES_PASSWORD" \
  --pg-database myapp
```

`serve` preflights upstream, opens `PGlite::open_multi_process`, starts a logical replica, scans schema, starts CDC, and binds HTTP.

## 3. Query a snapshot

```bash
curl -i -X POST http://127.0.0.1:8080/query \
  -H "content-type: application/json" \
  -d '{"sql":"select id, email from users where id = 7"}'
```

Public query response:

```http
HTTP/1.1 303 See Other
Location: /q/{hash}/{version}
Cache-Control: no-store
```

Fetch the snapshot:

```bash
curl http://127.0.0.1:8080/q/{hash}/{version}
```

## 4. Subscribe live

```bash
curl -N -X POST "http://127.0.0.1:8080/query?live=true" \
  -H "content-type: application/json" \
  -d '{"sql":"select id, status from orders"}'
```
