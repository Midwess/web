---
title: SQL classifier
description: Rules for SQL accepted by PgPaw.
---

PgPaw accepts one read-only PostgreSQL `SELECT` statement over replicated tables.

## Rejected

- Parse failures.
- Multiple statements.
- Writes and DDL.
- `SELECT INTO`.
- Locking reads such as `FOR UPDATE` and `FOR SHARE`.
- References to tables not present in the replica.
- Side-effecting functions: `nextval`, `setval`, advisory lock/unlock functions.
- Non-deterministic reads: `now`, `random`, `random_normal`, `clock_timestamp`, `timeofday`, `statement_timestamp`, `gen_random_uuid`, `uuid_generate_v4`.

## Extracted metadata

For accepted SQL, `ReadClassifier` returns:

| Field | Meaning |
| --- | --- |
| `fingerprint` | `DefaultHasher` of the parsed statement string. |
| `tables` | Lowercase table names from relation visits, excluding CTE aliases. |
| `eq_filters` | Literal equality filters from top-level `AND` expressions. |
| `sql` | Original SQL string. |

Equality filters are used to choose narrower version watermarks when possible.

## Example accepted query

```sql
select id, email
from users
where id = 7
```

## Example rejected query

```sql
select nextval('orders_id_seq')
```
