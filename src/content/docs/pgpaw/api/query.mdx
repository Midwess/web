---
title: POST /query
description: Run a read-only SELECT and receive a redirect or private inline JSON.
---

`POST /query` accepts:

```json
{
  "sql": "select id, email from users where id = 7"
}
```

## Public response

When every touched table is public, PgPaw materializes the query and returns:

```http
HTTP/1.1 303 See Other
Location: /q/{hash}/{version}
Cache-Control: no-store
```

The response body is empty. Follow `Location` to fetch the JSON snapshot.

## Private response

When any table has RLS enabled or lacks `PUBLIC SELECT`, PgPaw requires a bearer token. It verifies the token, extracts the configured role claim, runs the query through `query_json_as`, and returns:

```http
HTTP/1.1 200 OK
Content-Type: application/json
Cache-Control: private, no-store
```

## Snapshot identity

`hash` is the hexadecimal `DefaultHasher` of the parsed statement string. `version` is the `VersionIndex` LSN for touched tables and equality filters.
