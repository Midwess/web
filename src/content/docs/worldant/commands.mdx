title: Commands
description: The proposed immediate execution and explicit transaction model.
---

> This is the accepted target model. The runtime migration is not shipped yet.

A Command is Worldant's one immediate executable concept. It can read, write,
compute, call supported external services, and schedule a Workflow.

```text
apps/todo/commands/
  add.ts
  list.ts
```

The path defines the callable identity:

```text
apps/todo/commands/add.ts -> command: todo.add
```

## Commit boundaries

Standalone database operations commit independently. A later Command failure
does not undo operations that already completed.

Use an explicit transaction when multiple database operations must commit or
roll back together:

```ts
import { db, sql } from "worldant";

export default async function transfer(input: {
  from: string;
  to: string;
  amount: number;
}) {
  return db.transaction(async (tx) => {
    await tx.exec(sql`
      update accounts
      set balance = balance - ${input.amount}
      where id = ${input.from}
    `);
    await tx.exec(sql`
      update accounts
      set balance = balance + ${input.amount}
      where id = ${input.to}
    `);
  });
}
```

Only calls made through `tx` participate. Plain `db` calls remain independent,
even inside the callback. Keep external network calls outside database
transactions so they do not hold connections or locks.

## Commands and Workflows

```text
Command  -> immediate, non-durable execution
Workflow -> durable deterministic orchestration
Step     -> internal Workflow IO checkpoint
```

Query, Mutation, Reactive, and Action are removed as public executable concepts.
Reads and writes are Commands. Use a Workflow only when execution must survive
waiting, retries, process failure, or restart.
