---
title: Same origin
description: Consume Worldant from the same web origin.
---

Same-origin mode means the browser calls `/__worldant/v1` on the same origin as
the page. A proxy or router forwards that path to the Worldant daemon.

This avoids CORS and lets session cookies or same-origin credentials flow
through the normal application boundary.

The generated package still uses `worldant-client`; only the default `baseUrl`
changes.

```ts
import { configure, move } from "@world/chess";

configure({ baseUrl: "", prefix: "/__worldant/v1" });
await move({ gameId, from: "e2", to: "e4" }, { key: "move:e2:e4" });
```
