Worldant does not replace the Workflow compiler or framework adapters. It provides the world that those packages run against.

## Define functions

```ts
async function reserveInventory(sku: string) {
  "use step";
  return { sku, reserved: true };
}

export async function shipOrder(orderId: string, sku: string) {
  "use workflow";
  const reservation = await reserveInventory(sku);
  return { orderId, reservation };
}
```

`"use step"` marks side-effecting work that should be memoized by the workflow runtime. `"use workflow"` marks the durable function entrypoint.

## Register the world

```ts
import { install } from "@midwess/worldant";

await install({ id: "orders" });
```

For advanced setups, use `createWorld()` and register manually:

```ts
import { createWorld, setWorld } from "@midwess/worldant";

const world = await createWorld({ id: "orders" });
setWorld(world);
await world.start?.();
```

## Streams

Worldant exports `outputStream()` for streaming workflow results. The source code calls `getWritable()` from `workflow` and writes encoded chunks.

```ts
import { outputStream } from "@midwess/worldant";

async function emit(orderId: string) {
  "use step";
  const out = outputStream<{ state: string }>();
  await out.write({ state: "accepted" });
  await out.write({ state: "packed" });
  await out.close();
}
```

Call `outputStream()` from a step, because it performs I/O.

## Queues

The world implements queue creation and dispatch through `@workflow/world`. Queue handlers are registered by prefix. Native dispatch calls back into JS with the queue name, message, attempt, and message id metadata.
