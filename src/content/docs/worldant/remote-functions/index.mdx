Remote functions expose workflow entrypoints through `/__worldant/v1`. The workflow still runs in the durable world; the client only starts a run, waits or polls for completion, and decodes the result.

## Enable endpoint

```ts
import { createWorld } from "@midwess/worldant";

const world = await createWorld({
  id: "orders",
  net: true
});
await world.start?.();
```

Use an object for auth, CORS, or a dedicated port:

```ts
await createWorld({
  id: "@acme/orders",
  net: {
    token: process.env.WORLDANT_TOKEN,
    cors: ["https://app.example.com"],
    port: 4001
  }
});
```

## Generated package

A world can serve an npm package from:

```bash
npm install <world-id> --registry https://<host>/__worldant/v1/npm
```

The package includes:

- `worldName`
- `workflows`
- `defaultConfiguration`
- one typed wrapper per aliased workflow
- `configure(config)` for runtime connection overrides

`defaultConfiguration` contains public connection data only: world id, base URL, prefix, and polling interval. Put tokens in runtime config.
