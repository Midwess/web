## Versioning

`VersionIndex` consumes committed transactions from the replication bridge. It tracks the highest LSN relevant to a table or keyed row. A query version is the maximum relevant LSN for its tables and equality filters.

Snapshot key:

```txt
{hex(DefaultHasher(parsed_statement))}:{version}
```

When upstream data changes, the same SQL gets a new version. Old snapshot URLs keep serving their cached body until evicted.

## Live subscriptions

`LiveHub` stores the last result for each subscription. On each committed transaction that touches subscribed tables, it:

- re-runs the SQL against the replica;
- builds row maps using a primary key when known;
- diffs old and new rows;
- sends `insert`, `update`, `delete`, and `up-to-date` SSE frames.

If the internal broadcast channel lags, it sends `{"op":"reset"}` and clears subscriptions.

## Privacy

PgPaw asks the replica catalog whether any touched table has RLS enabled or lacks `PUBLIC SELECT`. If any table is private, the whole query is private.
